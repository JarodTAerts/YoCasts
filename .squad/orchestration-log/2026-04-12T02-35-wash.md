## Technical Decisions

1. **Cache-first + async fetch hybrid** — `makeWebRequest()` is async, but Menu2 builds items in constructor and can't update after. Sync getters return cached data (or empty arrays); async fetches populate cache and call `requestUpdate()`.
2. **`/user/login_pocket_casts` over `/user/login`** — Returns full OAuth2 token set (accessToken + refreshToken + expiresIn) vs legacy endpoint that returns only access token.
3. **Proactive token refresh** — Checks expiry before each API call. Falls back to full re-login if refresh fails (since `/user/token` is unreliable).
4. **On-demand episode fetch** — Episodes fetched per-podcast when user navigates to episode list, not bulk-loaded at startup.

## Known Limitations

- **Menu2 views can't dynamically add items after construction** — Episode list shows "Loading..." on first visit; user must back out and re-enter after fetch completes. Proper loading→populated transition needs view swap mechanism (future polish).
- **Token refresh race** — If refresh is in-flight when a data request fires, the data request uses the old token. Acceptable for v1 (tokens typically have 1hr lifetime).
